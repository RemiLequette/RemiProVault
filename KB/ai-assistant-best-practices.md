## Working with specs
[up](#table-of-contents)

When working on a project with spec or instruction files:
- When you are about to use a tool or change data, give a clear, concise, and descriptive plan and wait for my approval
- Read existing specs before coding or proposing a solution
- Consult the relevant spec to understand how to access information — do not infer it by digging through source files
- Update the spec before or at the same time as the code — never after
- Reference sources rather than duplicating their content
- If you should have consulted a spec before acting, say so — even after the fact

## Implementation order
[up](#table-of-contents)

For any implementation, follow this order:

**spec → test → code → run tests → debug**
